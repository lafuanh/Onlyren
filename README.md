# Onlyren

A full-stack room rental platform built with **Vue 3** (frontend) and **Laravel 12** (backend), containerised with Docker.  
Users can browse and book rooms; renters can list and manage properties; admins oversee the entire platform.

## Project Overview

**Onlyren** is an online marketplace for short- and long-term room rentals.

| Role | Capabilities |
|------|-------------|
| **Guest** | Browse rooms, view details, search/filter |
| **User** | Register/login, make reservations, process payments, message renters, manage profile |
| **Renter** | List rooms, manage availability, view bookings/payments, respond to messages |
| **Admin** | Full platform oversight: users, properties, orders, payments, reports, settings |

## Architecture & Design

### System Overview

```
Browser
  │
  ▼
Nginx (port 8080 dev / 80 prod)
  │  ├─ /              → Vue 3 SPA (static assets)
  │  └─ /api/*         → PHP-FPM (Laravel API)
  │
  ▼
MySQL 8 (port 3306)
```

### Main Components

| Component | Path | Role |
|-----------|------|------|
| **Frontend SPA** | `Onlyren_Frontend/` | Vue 3 single-page application |
| **REST API** | `Onlyren_Backend/` | Laravel 12 JSON API |
| **Nginx** | `nginx/dev.conf` | Reverse proxy & static file server |
| **Database** | MySQL 8 (Docker) | Persistent relational data store |

### Key Flows

**Authentication**
1. `POST /api/login` → Laravel Sanctum issues a personal access token.
2. Token stored client-side and sent as `Authorization: Bearer <token>` on subsequent requests.
3. Role-based middleware (`admin`, `renter`, `user`) restricts routes accordingly.

**Room Booking**
1. Guest searches rooms → `GET /api/search`.
2. User views room → `GET /api/rooms/{id}`.
3. User submits reservation → `POST /api/reservations`.
4. User processes payment → `POST /api/payments/reservation/{id}`.
5. Renter sees new booking in their dashboard.

## Getting Started

### Prerequisites

- [Docker](https://docs.docker.com/get-docker/) ≥ 24 and Docker Compose ≥ 2
- **OR**, for running without Docker:
  - PHP ≥ 8.2 with extensions: `pdo_mysql`, `mbstring`, `bcmath`, `gd`
  - Composer ≥ 2
  - Node.js ≥ 20 and npm ≥ 10
  - MySQL 8 (or SQLite for quick local dev)

### Installation

```bash
git clone https://github.com/lafuanh/Onlyren.git
cd Onlyren
```

### Configuration

1. Copy the backend environment file:

```bash
cp Onlyren_Backend/.env.example Onlyren_Backend/.env
```

2. Edit `Onlyren_Backend/.env` and set at minimum:

```dotenv
APP_KEY=          # generated by `php artisan key:generate`
APP_URL=http://localhost:8080

# SQLite (simplest for local dev — no extra service needed)
DB_CONNECTION=sqlite

# --- OR MySQL (matches Docker Compose setup) ---
# DB_CONNECTION=mysql
# DB_HOST=mysql          # service name inside Docker
# DB_PORT=3306
# DB_DATABASE=onlyrent_db
# DB_USERNAME=root
# DB_PASSWORD=rootpassword
```

> **Docker users**: the Docker Compose file already injects `APP_ENV`, `APP_DEBUG`, and links the `mysql` service — you only need to set `APP_KEY`.

### Running Locally (Docker)

```bash
# Start all services (frontend :3000, backend via nginx :8080, mysql :3306)
docker compose -f docker-compose.dev.yml up --build

# First-time database setup (in a new terminal)
docker compose -f docker-compose.dev.yml exec backend php artisan key:generate
docker compose -f docker-compose.dev.yml exec backend php artisan migrate --seed
```

| Service | URL |
|---------|-----|
| Frontend (HMR) | http://localhost:3000 |
| Backend API | http://localhost:8080/api |
| MySQL | localhost:3306 |

### Running Locally (Without Docker)

**Backend**

```bash
cd Onlyren_Backend

composer install

php artisan key:generate
php artisan migrate --seed

php artisan serve          # http://localhost:8000
```

**Frontend** (in a separate terminal)

```bash
cd Onlyren_Frontend

npm install

# The dev proxy in vite.config.js points to https://onlyren.noupal.pro by default.
# For local development, update the proxy target in vite.config.js:
#   target: 'http://localhost:8000'

npm run dev                # http://localhost:3000
```

### Building for Production

```bash
# Frontend — produces dist/
cd Onlyren_Frontend && npm run build

# Backend — cache config/routes/views for performance
cd Onlyren_Backend
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

## Usage

### Common Commands

#### Backend (Laravel Artisan)

```bash
# Database
php artisan migrate                  # Run pending migrations
php artisan migrate:fresh --seed     # Reset and seed
php artisan db:seed                  # Seed only

# Development
php artisan serve                    # Dev server (port 8000)
php artisan tinker                   # Interactive REPL

# Code quality
./vendor/bin/pint                    # Format PHP (Laravel Pint)
php artisan test                     # Run test suite
```

#### Frontend (npm)

```bash
npm run dev       # Start Vite dev server with HMR
npm run build     # Production build → dist/
npm run preview   # Preview production build locally
npm run lint      # ESLint auto-fix
npm run format    # Prettier format src/
```

### Example API Requests

```bash
# Register
curl -X POST http://localhost:8080/api/register \
  -H "Content-Type: application/json" \
  -d '{"name":"Alice","email":"alice@example.com","password":"secret","password_confirmation":"secret"}'

# Login
curl -X POST http://localhost:8080/api/login \
  -H "Content-Type: application/json" \
  -d '{"email":"alice@example.com","password":"secret"}'
# Returns: { "token": "<sanctum-token>", ... }

# List rooms (public)
curl http://localhost:8080/api/rooms

# Get user profile (authenticated)
curl http://localhost:8080/api/user/profile \
  -H "Authorization: Bearer <sanctum-token>"
```

## Testing

### Backend (PHPUnit)

```bash
cd Onlyren_Backend

# Run all tests
php artisan test

# Or directly with PHPUnit
./vendor/bin/phpunit

# Run a specific suite
./vendor/bin/phpunit --testsuite Unit
./vendor/bin/phpunit --testsuite Feature
```

Test files live in `Onlyren_Backend/tests/Unit/` and `tests/Feature/`.  
The `phpunit.xml` at the backend root defines both suites and sets `APP_ENV=testing`.

### Frontend (Lint / Format)

There is no automated frontend test runner configured yet.

```bash
cd Onlyren_Frontend

npm run lint      # ESLint with auto-fix
npm run format    # Prettier formatting check/fix
```

## Deployment

### Docker Production Build

```bash
# Build and start production containers
docker compose up --build -d
```

The default `docker-compose.yml` extends the dev compose file. For a standalone production deployment, build the individual service images:

```bash
# Backend (Apache-based)
docker build -t onlyren-backend ./Onlyren_Backend

# Frontend (Nginx-served static assets)
docker build --target production -t onlyren-frontend ./Onlyren_Frontend
```

### Environment Checklist

Before deploying to production:

- [ ] Set `APP_ENV=production` and `APP_DEBUG=false`
- [ ] Generate a secure `APP_KEY` (`php artisan key:generate`)
- [ ] Configure `DB_*` variables to point to your production MySQL instance
- [ ] Set `APP_URL` to your public domain
- [ ] Run `php artisan migrate --force`
- [ ] Run `php artisan config:cache && php artisan route:cache && php artisan view:cache`
- [ ] Configure `MAIL_*` variables for transactional email
- [ ] Set secure `MYSQL_ROOT_PASSWORD` in Docker Compose or your orchestrator secrets

## Troubleshooting

| Symptom | Likely cause | Fix |
|---------|-------------|-----|
| `php artisan` errors on startup | Missing `APP_KEY` | `php artisan key:generate` |
| `SQLSTATE[HY000] [2002]` | MySQL not running / wrong host | Check `DB_HOST`; for Docker use service name `mysql` |
| CORS errors in browser | Mismatched `APP_URL` or CORS config | Update `APP_URL` in `.env`; review `config/cors.php` |
| Frontend can't reach API | Proxy target mismatch | Update `vite.config.js` proxy `target` to match your backend URL |
| 500 error after deployment | Cached config stale | `php artisan config:clear && php artisan cache:clear` |
| `npm run dev` fails | Node version mismatch | Use Node 20 (`nvm use 20`) |
| Vendor directory missing | Composer not run | `cd Onlyren_Backend && composer install` |
| `node_modules` missing | npm not run | `cd Onlyren_Frontend && npm install` |
